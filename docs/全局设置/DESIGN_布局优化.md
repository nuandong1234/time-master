# 设置页面布局优化设计

## 概述

对设置页面进行布局和视觉优化，解决当前方案中"颜色不搭、布局乱"的问题。仅修改 `SettingsView.vue` 一个文件，不改动 settings store 和后端代码。

## 布局方案：两列网格

```
┌─────────────────────┬─────────────────────┐
│  系统                │  通用               │
│  ─────────────────  │  ─────────────────  │
│  开机自启    [🔘]   │  ☀️ 浅色            │
│  最小化到托盘 [🔘]  │  🌙 深色            │
│                      │  💻 跟随系统        │
├─────────────────────┴─────────────────────┤
│  数据管理                                   │
│  ─────────────────────────────────────────  │
│  导出                                      │
│  [导出 JSON]                                │
│                                            │
│  导入                                      │
│  从备份文件恢复数据，将替换现有所有数据     │
│  [选择文件并导入]                           │
│                                            │
│  数据存储路径                               │
│  C:\...time-master-data\  [打开文件夹]     │
│                                            │
│  [重置所有数据]                             │
│  清空所有数据，此操作不可恢复               │
│                                            │
└────────────────────────────────────────────┘
```

## 变更清单

| 项 | 之前 | 之后 |
|--------|------|------|
| 布局 | 单列滚动 | 两列网格（系统+通用并排，数据管理跨列整行） |
| 分区标题 | 彩色圆点+文字 | 纯文字`text-sm font-semibold` + 底部淡灰边框线 `border-b border-border/50` |
| 主题选择器 | 横排3按钮 inline-flex | 竖排3按钮 flex-col |
| 关于分区 | 独立分区 | 移除 |
| 版本号 | 底部居中 | 移除 |
| 危险操作框 | 红色边框 + 红色背景框 | 红色按钮 + 灰色提示文字（与导入样式一致） |
| 存储路径 | 路径文字一行 + 按钮一行 | 路径文字与按钮同一行（flex row） |
| 导出按钮颜色 | `bg-primary` 纯黑 | `bg-blue-600` 品牌蓝 |
| 次按钮样式 | `bg-secondary` 纯灰 | `border border-border bg-background` outline 样式 |

## 关键实现细节

### 布局
```
grid-template-columns: 1fr 1fr
gap: 24px
数据管理区: grid-column: 1 / -1
分区间距: mb-8（比当前 mb-10 紧凑）
```

### 按钮配色
- 主操作（导出）：蓝色 `#2563eb`，突出主要操作
- 次操作（导入/打开文件夹）：outline 灰边框，不抢眼
- 危险操作（重置）：红色 `#dc2626`

### 数据存储路径
```
flex-row, items-center, gap-2
<路径文字 flex-1> + <"打开文件夹"按钮 shrink-0>
```

### 重置区块
```
<红色按钮>
<灰色提示文字 12px>
```
与导入区块样式对称，去掉红色边框和背景。
