# 全局设置 — 共识文档

## 需求描述

为 Time-Master 桌面应用实现完整的全局设置功能。当前设置页（`SettingsView.vue`）为占位页，显示"即将推出..."。需替换为完整的设置界面，覆盖外观、数据管理、系统行为三大领域，并与现有番茄钟设置整合。

## 验收标准

| # | 标准 | 验证方式 |
|---|------|---------|
| 1 | 主题模式：浅色、深色、跟随系统三种模式可选，切换后即时生效 | 视觉验证 |
| 2 | 主题选择持久化，下次启动保持上次选择的主题 | 重启验证 |
| 3 | 数据导出为 JSON 格式 | 导出后手动打开文件验证 |
| 4 | 数据导出为 CSV 格式 | 导出后用 Excel 打开验证 |
| 5 | 导出范围：所有数据（事项清单 + 番茄钟 + 项目流程） | 导出文件内容验证 |
| 6 | 导入数据：完全替换（清空现有数据，用导入数据覆盖） | 导入前后数据对比 |
| 7 | 重置所有数据：带二次确认弹窗，确认后清空全部数据 | 操作验证 |
| 8 | 开机自启：默认关闭，用户可手动开启/关闭 | 设置后重启电脑验证 |
| 9 | 开机自启时最小化到系统托盘，不弹出窗口 | 开机后验证 |
| 10 | 关闭窗口按钮（X）行为：最小化到系统托盘，不退出应用 | 点击 X 验证 |
| 11 | 系统托盘图标：使用现有 icon.ico | 外观验证 |
| 12 | 托盘右键菜单：显示"显示主窗口"、"退出应用"两个选项 | 右键点击验证 |
| 13 | 番茄钟完成时弹出系统通知（全部轮次完成 + 每轮切换，两者都要） | 番茄钟运行验证 |

## 技术约束

- 前端框架：Vue 3 + TypeScript + Vite
- 样式方案：Tailwind CSS 4 + CSS 变量（已有 `.dark` 暗色模式变量）
- 数据持久化：Tauri 文件读写命令（JSON 文件）
- 桌面框架：Tauri v2
- 系统托盘 / 开机自启：需安装 Tauri 插件（`tauri-plugin-tray`、`tauri-plugin-autostart`）

## 与现有系统集成

- 主题切换：修改 `<html>` 或 `<body>` 的 class，利用现有 `.dark` CSS 变量
- 番茄钟设置：`focusMinutes`、`breakMinutes`、`totalRounds` 已有 store 方法
- 数据管理：复用 `dataStore`（`TauriDataStore`）的 `read`/`write` 方法
- 设置持久化：新增 `data/settings.json` 存储设置数据
- 系统通知：已有 `send_native_notification` Rust 命令 + `tauri-plugin-notification`
