# Domain Docs

工程技能在探索代码库时，应按以下方式消费本仓库的领域文档。

## 探索前先阅读

- **`CONTEXT.md`** — 位于仓库根目录
- **`docs/adr/`** — 读取与你即将修改区域相关的架构决策记录

如果这些文件不存在，**直接跳过**。不要提示缺失，也不要建议创建。`/domain-modeling` 技能会在需要时按需创建。

## 文件结构

单一上下文仓库：

```
/
├── CONTEXT.md
├── docs/adr/
│   └── ...
└── src/
```

## 使用词汇表中的术语

当输出命名领域概念时（问题标题、重构建议、假设、测试名称），使用 `CONTEXT.md` 中定义的术语。不要使用词汇表明确避免的同义词。

如果需要的概念不在词汇表中，这是一个信号——要么你在发明项目不使用的语言（重新考虑），要么确实存在空白（交给 `/domain-modeling` 处理）。

## 标记 ADR 冲突

如果你的输出与现有 ADR 矛盾，显式标明：

> _与 ADR-0007（事件溯源订单）矛盾——因为……值得重新讨论_
