# PRD: 设置页面布局优化

## Problem Statement

当前设置页面布局为单列滚动，分区标题使用彩色圆点装饰（蓝色/黄色/绿色/灰色），颜色杂乱且与内容无实际关联。整体布局松散，按钮颜色不协调（纯黑/纯灰/红色），视觉层次不够清晰。用户需要更干净、更紧凑、色彩更协调的设置界面。

## Solution

将设置页面从单列滚动改为两列网格布局。系统设置与通用设置并排排列，数据管理跨列整行。去掉所有彩色圆点装饰，分区标题改用纯文字加粗加底部淡灰边框线。统一按钮配色方案：主操作用品牌蓝色、次操作用 outline 灰边框样式、危险操作用红色。缩小危险操作区块，去掉红色边框和背景框，改为红色按钮加灰色提示文字。去掉"关于"分区和版本号。

## User Stories

1. 作为用户，我想要设置页面布局更紧凑，以便在更少的滚动中看到所有设置项
2. 作为用户，我想要分区标题不用彩色圆点装饰，以便界面更干净整洁
3. 作为用户，我想要系统与通用设置并排显示，以便充分利用横向空间
4. 作为用户，我想要主题选择按钮竖排显示，以便在窄列中完整显示
5. 作为用户，我想要数据存储路径与打开按钮在同一行，以便减少垂直占用
6. 作为用户，我想要重置所有数据的按钮更紧凑，以便不占用过多空间
7. 作为用户，我想要导出按钮使用醒目的蓝色，以便快速找到主要操作
8. 作为用户，我想要导入和打开文件夹按钮使用 outline 样式，以便与主要操作区分层次
9. 作为用户，我想要去掉关于分区和版本号，以便只保留真正需要的设置项

## Implementation Decisions

### 布局架构
- 使用 CSS Grid 实现两列布局：`grid-template-columns: 1fr 1fr`，间距 24px
- 数据管理区使用 `grid-column: 1 / -1` 跨列整行
- 分区间距从 `mb-10` 缩小为 `mb-8`

### 分区标题
- 去掉彩色圆点 `<span class="size-2 rounded-full bg-xxx">`
- 改为纯文字 `text-sm font-semibold text-foreground` 加底部 `border-b border-border/50`

### 主题选择器
- 从 `inline-flex` 横排改为 `flex flex-col` 竖排
- 按钮样式保持不变（选中时高亮）

### 数据存储路径
- 从两行（路径文字 + 按钮）改为 `flex-row items-center gap-2` 同一行
- 路径文字 `flex-1`，按钮 `shrink-0`

### 危险操作区块
- 去掉红色边框 `.border-red-200` 和红色背景框
- 改为红色按钮 + 下方灰色提示文字，与导入区块样式对齐

### 按钮配色
- 导出按钮：`bg-blue-600 text-white`（品牌蓝色）
- 次按钮：`border border-border bg-background`（outline 灰色边框）
- 危险按钮：`bg-red-500 text-white`（红色）

### 删除内容
- 移除"关于"分区（version + tech stack）
- 移除页面底部版本号

### 修改范围
- 仅修改设置页面视图一个文件
- 不动 settings store，不动后端代码

## Testing Decisions

当前项目没有前端测试基础设施。本次改动仅涉及 UI 布局和样式，不涉及业务逻辑。验证方式为手动视觉检查：

1. 在两列布局下，系统与通用并排显示正常
2. 主题选择器竖排显示，三个选项都完整可见
3. 数据存储路径与按钮在同一行，宽屏下不换行
4. 重置区块紧凑显示，无红色边框
5. 深色模式下所有颜色正常工作
6. 独立设置窗口（640px 宽）下两列不溢出

## Out of Scope

- 不修改 settings store 的数据结构
- 不修改后端 Rust 代码
- 不修改其他页面的布局
- 不添加新功能或新设置项
- 不涉及单元测试或 e2e 测试
